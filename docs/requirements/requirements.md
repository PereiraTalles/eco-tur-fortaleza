# Requisitos do Sistema

Este documento descreve os requisitos do sistema EcoTur Fortaleza, desenvolvido na disciplina Projeto Aplicado Multiplataforma (Etapa 2).  
Os requisitos foram definidos com base no problema identificado e no objetivo de apoiar o ODS 11 (Cidades e Comunidades Sustentáveis).

Abaixo estão listados os requisitos funcionais e não funcionais do sistema.

## Requisitos Funcionais

1. O sistema deve permitir listar pontos turísticos ecológicos.  
2. O usuário deve poder filtrar os pontos por categoria.  
3. O usuário deve poder filtrar os pontos por bairro/localização.  
4. O sistema deve permitir buscar pontos pelo nome.  
5. O sistema deve exibir detalhes de cada ponto turístico.  
6. O frontend deve consumir a API do backend para buscar os dados.  
7. O backend deve retornar os dados em formato JSON.  

## Requisitos Não Funcionais

1. O sistema deve ser fácil de navegar e intuitivo.  
2. O tempo de resposta da API deve ser curto para garantir boa experiência ao usuário.  
3. O sistema deve funcionar em navegadores modernos (Chrome, Firefox, Edge).  
4. O backend deve manter uma estrutura organizada, seguindo boas práticas de código.  
5. O banco de dados deve garantir integridade e consistência dos dados.  
6. O sistema deve ser hospedado em ambientes confiáveis (Vercel e Render).  
7. O código deve ser versionado no GitHub.
